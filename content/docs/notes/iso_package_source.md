## Install packages from ISO installation image

## DNF

### Fedora

- Set gpgkey path appropriately for the running version.
- Or if you can trust the source, ignore it with gpgcheck=0.

```bash
mkdir -p /mnt/iso
mount -oro /path/to/iso /mnt/iso
tee /etc/yum.repos.d/iso.repo <<EOF
[iso]
name=iso
baseurl=file:///mnt/iso
enabled=1
gpgkey=file:///etc/pki/rpm-gpg/RPM-GPG-KEY-44-fedora
metadata_expire=-1
EOF
```

### Red Hat Enterprise Linux

- Gpgkey paths are common accross versions.
- Or if you can trust the source, ignore it with gpgcheck=0.

```bash
mkdir -p /mnt/iso
mount -oro /path/to/iso /mnt/iso
tee /etc/yum.repos.d/iso.repo <<EOF
[iso-baseos]
name=iso-baseos
baseurl=file:///mnt/iso/BaseOS
enabled=1
gpgkey=file:///etc/pki/rpm-gpg/RPM-GPG-KEY-redhat-release
metadata_expire=-1
[iso-appstream]
name=iso-appstream
baseurl=file:///mnt/iso/AppStream
enabled=1
gpgkey=file:///etc/pki/rpm-gpg/RPM-GPG-KEY-redhat-release
metadata_expire=-1
EOF
```

## APT

- No appropriate GPG keys for ISO are included unless I'm missing something.

- Set `Signed-By` to some GPG key or empty, and set `Trusted` yes to skip GPG verification.

- Suites `stable` is a symlink and works fine on Debian, works with warnings on Ubuntu.
  `ll /mnt/iso/dists` to see available Suite names and symlinks.

- It seems like `apt modernize-sources` generates `file:/mnt/iso` URI and it works.
  However, it seems to fail to migrate CD-ROM URI generated by `apt-cdrom`.
  This is why I dicided to use more general `file:///` URI in below examples.

### Debian

```bash
mkdir -p /mnt/iso
mount -oro /path/to/iso /mnt/iso
tee /etc/apt/sources.list.d/iso.sources <<EOF
Types: deb
URIs: file:///mnt/iso
Suites: stable
Components: main non-free-firmware
Signed-By: /usr/share/keyrings/debian-archive-keyring.gpg
Trusted: yes
EOF
```

### Ubuntu

```bash
mkdir -p /mnt/iso
mount -oro /path/to/iso /mnt/iso
tee /etc/apt/sources.list.d/iso.sources <<EOF
Types: deb
URIs: file:///mnt/iso
Suites: stable
Components: main restricted
Signed-By: /usr/share/keyrings/ubuntu-archive-keyring.gpg
Trusted: yes
EOF
```

## Link

For offline update, `dnf download` and `apt --download-only` is the choice.

[apt-offline(8)](https://manpages.debian.org/unstable/apt-offline/apt-offline.8.en.html)\
`apt-offline` can install packages on offline machine.
Generate a signature file on offline machine, download packages metadata on online machine using the signature file,
and install packages metadata on offline machine.
Do the same iteration with packages file.

[AptGet/Offline - Community Help Wiki](https://help.ubuntu.com/community/AptGet/Offline)\
for offline repository metadata like apt-offline's first phase.
